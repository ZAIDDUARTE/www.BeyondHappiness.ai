# Beyond Happiness Chatbot

This project is a chatbot website designed to help people manage stress and improve emotional well-being. It offers a responsive, user-friendly interface with a focus on accessibility and includes dark mode support. Users can interact with the chatbot to get personalized advice on managing stress, making use of integrated APIs for handling user data and providing real-time feedback.

## Technologies Used
- **Tailwind CSS**: Styling is implemented using the Tailwind CSS framework with a CDN link:
   ```html
   <script src="https://cdn.tailwindcss.com"></script>
   ```
- **Live Server Plugin**: The project can be run using the Live Server plugin in VSCode or any similar tool that serves static files locally.

## Features
- **Responsive Design**: The website is fully responsive and adapts to different screen sizes.
- **Dark Mode**: Support for a dark theme to enhance user experience in low-light environments.
- **API Integration**: Images and certain data are uploaded via a server's API, ensuring a dynamic user experience.
- **Commented Code**: Each file contains comments to help understand the structure and functionality of each section.

## Project Structure
The project is organized as follows:

```plaintext
├── chat/               # Contains the main chat page files.
├── theme/              # Contains dark mode page files.
├── main/               # Includes the main pages and upgrade pages.
├── settings/           # Contains settings section files.
├── assets/             # Holds images used in the project.
```

## Installation & Running the Project
To run this project locally, follow these steps:

1. Clone the repository or download the source files.
2. Ensure you have a code editor like **VSCode** installed.
3. Install the **Live Server** plugin for VSCode (or any similar tool).
4. Open the project folder in your editor and start the server via Live Server to view the website in your browser.

> Note: Since this is a static front-end project, no additional dependencies are required beyond a live server for local development.
